## 1. 产品概述

### 1.1 产品背景

在信息爆炸的时代，用户需要一个简洁高效的个人管理工具。现有的解决方案往往功能分散，用户需要在多个APP之间切换：新闻阅读、笔记记录、财务管理分别使用不同的应用。KINETIC_NOTES 旨在整合这些高频使用场景，提供一站式的个人信息管理体验，并采用独特的 Neo-Brutalist 设计风格，打造差异化视觉体验。

### 1.2 产品定位

KINETIC_NOTES 定位为"轻量级个人信息管理中心"，面向追求简洁高效生活方式、同时欣赏独特设计美学的年轻用户群体。产品特点是：
- **功能聚焦：** 只做用户最需要的四件事（新闻、笔记、收入、支出）
- **设计独特：** Neo-Brutalist 设计风格，高对比度、锐利边缘、大胆配色
- **体验简洁：** 极简设计，无广告干扰
- **数据本地：** 用户数据优先本地存储，保护隐私

### 1.3 设计风格

**Neo-Brutalist（新粗野主义）**
- 高对比度、锐利边缘
- 4px 粗边框
- 倾斜元素设计
- 暗色背景 (#131313)
- 双强调色：金黄色 (#FFD700) + 青色 (#007F7F)
- 无圆角设计（0px）

### 1.4 目标用户

**主要用户：** 18-35岁的年轻白领、学生、自由职业者

**用户画像：**

| 特征 | 描述 |
|------|------|
| 年龄 | 18-35岁 |
| 职业 | 白领/学生/自由职业者 |
| 技术水平 | 中等，熟悉智能手机操作 |
| 使用场景 | 晨间阅读、随手记录、月底记账 |
| 核心诉求 | 一站式管理、独特设计、快速操作 |
| 痛点 | 多APP切换繁琐、记账工具复杂、新闻APP广告多 |

### 1.5 产品目标

- **业务目标：** 上线3个月内达到10万活跃用户，日留存率≥40%
- **用户目标：** 减少APP切换次数，提升信息管理效率50%
- **技术目标：** 页面加载≤1秒，支持离线使用

## 2. 用户场景

### 2.1 用户旅程地图

**场景1：晨间阅读**
- **用户：** 上班通勤的白领小明
- **目标：** 在地铁上快速浏览今日热点新闻
- **步骤：**
  1. 打开APP，自动进入新闻页面
  2. 滑动浏览新闻列表（Neo-Brutalist 卡片风格）
  3. 点击感兴趣的新闻查看详情
  4. 需要时收藏到记事本
- **痛点：** 传统新闻APP广告多、加载慢、资讯杂乱
- **解决方案：** 精选高质量新闻源，无广告设计，独特视觉风格

**场景2：随手记录**
- **用户：** 会议中需要记录要点的小明
- **目标：** 快速记录会议关键信息，事后整理
- **步骤：**
  1. 打开APP，切换到记事本（点击FAB黄色添加按钮）
  2. 进入编辑页，输入标题和内容
  3. 选择分类和标签（带旋转角度的标签设计）
  4. 保存笔记，返回Bento Grid布局的笔记列表
- **痛点：** 传统笔记APP打开慢、功能复杂、找不到之前记录
- **解决方案：** 快速入口、Bento Grid布局、归档搜索功能

**场景3：月底记账**
- **用户：** 月底查看本月收支的小明
- **目标：** 了解本月收入支出情况，规划下月预算
- **步骤：**
  1. 打开APP，切换到财务模块
  2. 查看本月收支概览（Neo风格数字卡片）
  3. 按分类查看支出详情
  4. 添加遗漏的收支记录
- **痛点：** 传统记账APP操作繁琐、分类复杂、图表花哨不实用
- **解决方案：** 简洁统计、快速录入、实用分类

## 3. 功能需求

### 3.1 核心功能（P0 - MVP必需）

#### 3.1.1 新闻模块

**功能描述：**
提供精选新闻内容，用户可浏览、阅读、收藏新闻文章。

**用户价值：**
一站式获取高质量资讯，无需额外下载新闻APP。

**功能流程：**
```
打开APP → 自动加载新闻列表 → 用户浏览/点击 → 查看详情 → 可选收藏到记事本
```

**详细规则：**
- 规则1：新闻列表默认显示10条，下拉加载更多
- 规则2：新闻来源为精选高质量媒体（API获取）
- 规则3：支持按时间倒序排列
- 规则4：点击新闻进入详情页，支持返回
- 规则5：新闻详情页可一键收藏到记事本

**界面要求（Neo-Brutalist风格）：**
- **页面布局：**
  - 顶部：倾斜字体品牌标识 + 刷新按钮（黄色高亮）
  - 中部：新闻卡片网格（Bento Grid布局）
  - 底部：底部导航栏
- **卡片样式：**
  - 0px圆角（无圆角）
  - 4px白色边框
  - 8px偏移阴影（Neo阴影效果）
  - Hover时卡片上移(-translate-y-1, -translate-x-1)
- **交互方式：** 点击浏览，下拉刷新，上拉加载更多

**数据要求：**
- **输入：** 新闻API请求
- **输出：** 新闻列表、新闻详情
- **存储：** 用户收藏的新闻ID

**异常处理：**
- 情况1：网络异常 → 显示本地缓存新闻 + 提示"离线模式"
- 情况2：API失败 → 显示"暂无新闻，稍后重试"

**验收标准：**
- [ ] 标准1：新闻列表加载时间≤2秒
- [ ] 标准2：下拉刷新功能正常
- [ ] 标准3：点击新闻可进入详情页
- [ ] 标准4：离线时可显示缓存新闻
- [ ] 标准5：卡片Hover上移效果正常

#### 3.1.2 记事本模块（my_notes）

**功能描述：**
提供笔记创建、编辑、删除、分类管理、归档搜索功能。

**用户价值：**
随手记录重要信息，分类管理便于查找，归档功能管理历史记录。

**功能流程：**
```
新建笔记 → 输入内容 → 选择分类标签 → 保存 → Bento Grid展示 → 可编辑/删除/归档
```

**详细规则：**
- 规则1：支持创建文本笔记、任务清单、图片笔记
- 规则2：每条笔记包含标题、内容、分类、标签、创建时间
- 规则3：预设分类：Work、Personal、Ideas、Tasks
- 规则4：笔记列表采用Bento Grid布局，响应式1/2/3列
- 规称5：支持编辑和删除笔记
- 规则6：支持全文搜索和归档管理
- 规则7：标签支持旋转角度设计（-rotate-3 ~ rotate-3）

**界面要求（Neo-Brutalist风格）：**

**顶部导航栏（TopAppBar）：**
- 倾斜字体的"KINETIC_NOTES"标题（黄色、粗体）
- 菜单按钮：侧边栏展开/收起控制
- 搜索按钮：快速跳转搜索页
- 用户头像：个人资料入口
- 固定顶部，4px白色底部边框

**侧边栏导航（桌面端 NavigationDrawer）：**
- 分类列表：Work、Personal、Ideas、Tasks
- 存储容量指示器：显示存储使用百分比
- 当前选中分类高亮显示
- 左侧固定，青色阴影

**笔记卡片网格（Bento Grid）：**
- 响应式布局：Mobile单列、Tablet双列、Desktop三列
- **卡片类型：**
  - 文本笔记卡片
  - 图片笔记卡片
  - 任务清单卡片
  - 引用/名言卡片
- **卡片信息：**
  - 分类标签（带旋转角度）
  - 创建/更新时间
  - 标题
  - 内容摘要
  - 附件指示图标
- **交互效果：**
  - Hover时卡片上移(-translate-y-1, -translate-x-1)
  - Neo-Brutalist阴影效果

**筛选与排序：**
- Filter按钮：打开筛选面板
- Sort By按钮：排序方式切换

**创建笔记FAB：**
- 固定在右下角的大号添加按钮
- 黄色背景(#FFD700)、白色边框、Neo阴影
- 点击进入编辑页

**移动端底部导航（BottomNavBar）：**
- 四个导航项：Notes、Tags、Archive、Settings
- 当前页面高亮显示（上移+阴影效果）
- 4px白色顶部边框

**数据要求：**
- **输入：** 笔记标题、内容、分类、标签
- **输出：** 笔记列表、笔记详情
- **存储：** 笔记数据本地存储（SQLite/IndexedDB）

**数据模型：**
| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | string | 是 | UUID | 笔记唯一ID |
| title | string | 是 | - | 笔记标题 |
| content | text | 是 | - | 笔记内容 |
| category | string | 是 | "Tasks" | 分类标签 |
| tags | array | 否 | [] | 标签列表 |
| card_type | string | 否 | "text" | 卡片类型：text/image/task/quote |
| card_color | string | 否 | "default" | 卡片颜色 |
| is_archived | boolean | 否 | false | 是否归档 |
| is_pinned | boolean | 否 | false | 是否置顶 |
| created_at | datetime | 是 | NOW() | 创建时间 |
| updated_at | datetime | 是 | NOW() | 更新时间 |

**异常处理：**
- 情况1：标题为空 → 提示"请输入标题"
- 情况2：内容为空 → 提示"请输入内容"

**验收标准：**
- [ ] 标准1：可成功创建、编辑、删除笔记
- [ ] 标准2：笔记列表按时间倒序显示
- [ ] 标准3：分类筛选功能正常
- [ ] 标准4：搜索功能可查找笔记
- [ ] 标准5：卡片Hover上移效果正常
- [ ] 标准6：侧边栏导航功能正常
- [ ] 标准7：响应式布局正确切换

## 5. 交互设计

### 5.1 用户交互流程

**流程1：创建笔记流程**
```
点击FAB添加按钮 → 进入编辑页 → 输入标题和内容 → 选择分类和标签 → 设置提醒（可选）→ 设置外观（可选）→ 点击保存 → 返回笔记列表
```

**流程2：浏览笔记流程**
```
主页查看Bento Grid笔记网格 → 使用侧边栏切换分类 → 使用筛选/排序调整视图 → 点击卡片查看详情 → Hover卡片上移预览
```

**流程3：搜索归档流程**
```
点击搜索图标或Archive导航 → 输入搜索查询 → 使用筛选芯片细化 → Hover卡片预览恢复状态 → 点击恢复按钮 → 笔记恢复到列表
```

**流程4：管理分类标签流程**
```
进入Tags页面 → 查看分类列表 → Hover显示编辑/删除按钮 → 编辑/删除分类 → 管理标签云 → 创建新分类或标签
```

### 5.2 状态转换

**笔记状态：**
| 当前状态 | 触发条件 | 目标状态 | 说明 |
|---------|---------|---------|------|
| 无 | 点击FAB | 编辑中 | 创建新笔记 |
| 编辑中 | 点击保存 | 已保存 | 笔记保存成功 |
| 编辑中 | 点击取消 | 无 | 取消创建 |
| 已保存 | 点击编辑 | 编辑中 | 进入编辑模式 |
| 已保存 | 长按归档 | 已归档 | 笔记移入归档 |
| 已归档 | 点击恢复 | 已保存 | 笔记恢复 |
| 已保存 | 长按删除 | 已删除 | 笔记删除成功 |

### 5.3 操作反馈

- **成功反馈：** 操作完成后显示Toast提示，黄色背景，1秒后消失
- **失败反馈：** 显示错误提示弹窗，青色边框，说明失败原因
- **加载反馈：** 数据加载时显示Neo风格loading动画
- **Hover反馈：** 卡片上移(-translate-y-1, -translate-x-1) + 阴影变化
