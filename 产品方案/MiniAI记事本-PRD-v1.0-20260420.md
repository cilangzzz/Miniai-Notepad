## 1. 产品概述

### 1.1 产品背景

在信息爆炸的时代，用户需要一个简洁高效的个人管理工具。现有的解决方案往往功能分散，用户需要在多个APP之间切换：新闻阅读、笔记记录、财务管理分别使用不同的应用。MiniAI记事本旨在整合这些高频使用场景，提供一站式的个人信息管理体验。

### 1.2 产品定位

MiniAI记事本定位为"轻量级个人信息管理中心"，面向追求简洁高效生活方式的年轻用户群体。产品特点是：
- 功能聚焦：只做用户最需要的四件事（新闻、笔记、收入、支出）
- 体验简洁：极简设计，无广告干扰
- 数据本地：用户数据优先本地存储，保护隐私

### 1.3 目标用户

**主要用户：** 18-35岁的年轻白领、学生、自由职业者

**用户画像：**

| 特征 | 描述 |
|------|------|
| 年龄 | 18-35岁 |
| 职业 | 白领/学生/自由职业者 |
| 技术水平 | 中等，熟悉智能手机操作 |
| 使用场景 | 晨间阅读、随手记录、月底记账 |
| 核心诉求 | 一站式管理、简洁界面、快速操作 |
| 痛点 | 多APP切换繁琐、记账工具复杂、新闻APP广告多 |

### 1.4 产品目标

- **业务目标：** 上线3个月内达到10万活跃用户，日留存率≥40%
- **用户目标：** 减少APP切换次数，提升信息管理效率50%
- **技术目标：** 页面加载≤1秒，支持离线使用

## 2. 用户场景

### 2.1 用户旅程地图

**场景1：晨间阅读**
- **用户：** 上班通勤的白领小明
- **目标：** 在地铁上快速浏览今日热点新闻
- **步骤：**
  1. 打开APP，自动进入新闻页面
  2. 滑动浏览新闻列表
  3. 点击感兴趣的新闻查看详情
  4. 需要时收藏到记事本
- **痛点：** 传统新闻APP广告多、加载慢、资讯杂乱
- **解决方案：** 精选高质量新闻源，无广告设计，秒级加载

**场景2：随手记录**
- **用户：** 会议中需要记录要点的小明
- **目标：** 快速记录会议关键信息，事后整理
- **步骤：**
  1. 打开APP，切换到记事本
  2. 点击新建笔记
  3. 输入/语音输入内容
  4. 选择分类保存
- **痛点：** 传统笔记APP打开慢、功能复杂、找不到之前记录
- **解决方案：** 快速入口、分类管理、智能搜索

**场景3：月底记账**
- **用户：** 月底查看本月收支的小明
- **目标：** 了解本月收入支出情况，规划下月预算
- **步骤：**
  1. 打开APP，切换到财务模块
  2. 查看本月收支概览
  3. 按分类查看支出详情
  4. 添加遗漏的收支记录
- **痛点：** 传统记账APP操作繁琐、分类复杂、图表花哨不实用
- **解决方案：** 简洁统计、快速录入、实用分类

## 3. 功能需求

### 3.1 核心功能（P0 - MVP必需）

#### 3.1.1 新闻模块

**功能描述：**
提供精选新闻内容，用户可浏览、阅读、收藏新闻文章。

**用户价值：**
一站式获取高质量资讯，无需额外下载新闻APP。

**功能流程：**
```
打开APP → 自动加载新闻列表 → 用户浏览/点击 → 查看详情 → 可选收藏到记事本
```

**详细规则：**
- 规则1：新闻列表默认显示10条，下拉加载更多
- 规则2：新闻来源为精选高质量媒体（API获取）
- 规则3：支持按时间倒序排列
- 规则4：点击新闻进入详情页，支持返回
- 规则5：新闻详情页可一键收藏到记事本

**界面要求：**
- **页面布局：**
  - 顶部：日期显示 + 刷新按钮
  - 中部：新闻卡片列表（标题+摘要+来源+时间）
  - 底部：导航栏
- **交互方式：** 点击浏览，下拉刷新，上拉加载更多
- **视觉设计：** 卡片式设计，白色背景，简洁排版

**数据要求：**
- **输入：** 新闻API请求
- **输出：** 新闻列表、新闻详情
- **存储：** 用户收藏的新闻ID

**异常处理：**
- 情况1：网络异常 → 显示本地缓存新闻 + 提示"离线模式"
- 情况2：API失败 → 显示"暂无新闻，稍后重试"

**验收标准：**
- [ ] 标准1：新闻列表加载时间≤2秒
- [ ] 标准2：下拉刷新功能正常
- [ ] 标准3：点击新闻可进入详情页
- [ ] 标准4：离线时可显示缓存新闻

#### 3.1.2 记事本模块

**功能描述：**
提供笔记创建、编辑、删除、分类管理功能。

**用户价值：**
随手记录重要信息，分类管理便于查找。

**功能流程：**
```
新建笔记 → 输入内容 → 选择分类 → 保存 → 列表展示 → 可编辑/删除
```

**详细规则：**
- 规则1：支持创建文本笔记
- 规则2：每条笔记包含标题、内容、分类、创建时间
- 规则3：预设分类：工作、生活、学习、其他（可自定义）
- 规则4：笔记列表按时间倒序排列
- 规则5：支持编辑和删除笔记
- 规则6：支持全文搜索笔记内容

**界面要求：**
- **页面布局：**
  - 顶部：搜索框 + 分类筛选
  - 中部：笔记卡片列表（标题+摘要+时间+分类标签）
  - 底部：新建按钮 + 导航栏
- **交互方式：** 点击查看详情，长按删除，点击新建
- **视觉设计：** 卡片式设计，分类用不同颜色标签区分

**数据要求：**
- **输入：** 笔记标题、内容、分类
- **输出：** 笔记列表、笔记详情
- **存储：** 笔记数据本地存储（SQLite/IndexedDB）

**数据模型：**
| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | string | 是 | 自动生成 | 笔记唯一ID |
| title | string | 是 | - | 笔记标题 |
| content | string | 是 | - | 笔记内容 |
| category | string | 是 | "其他" | 分类标签 |
| created_at | datetime | 是 | 当前时间 | 创建时间 |
| updated_at | datetime | 是 | 当前时间 | 更新时间 |

**异常处理：**
- 情况1：标题为空 → 提示"请输入标题"
- 情况2：内容为空 → 提示"请输入内容"

**验收标准：**
- [ ] 标准1：可成功创建、编辑、删除笔记
- [ ] 标准2：笔记列表按时间倒序显示
- [ ] 标准3：分类筛选功能正常
- [ ] 标准4：搜索功能可查找笔记

## 5. 交互设计

### 5.1 交互流程

**流程1：添加支出记录**
```
点击添加按钮 → 弹出输入表单 → 输入金额 → 选择分类 → 输入日期 → 可选备注 → 点击保存 → 返回列表
  ↓                                                      ↓
取消 → 关闭表单                                        保存失败 → 提示错误信息
```

**流程2：创建笔记**
```
点击新建按钮 → 进入编辑页 → 输入标题 → 输入内容 → 选择分类 → 点击保存 → 返回列表
```

### 5.2 状态转换

**笔记状态：**
| 当前状态 | 触发条件 | 目标状态 | 说明 |
|---------|---------|---------|------|
| 无 | 点击新建 | 编辑中 | 创建新笔记 |
| 编辑中 | 点击保存 | 已保存 | 笔记保存成功 |
| 编辑中 | 点击取消 | 无 | 取消创建 |
| 已保存 | 点击编辑 | 编辑中 | 进入编辑模式 |
| 已保存 | 长按删除 | 已删除 | 笔记删除成功 |

### 5.3 操作反馈

- **成功反馈：** 操作完成后显示Toast提示"保存成功"/"删除成功"，1秒后消失
- **失败反馈：** 显示错误提示弹窗，说明失败原因
- **加载反馈：** 数据加载时显示loading动画

## 6. 数据需求

### 6.1 数据模型

#### 实体1：笔记（Note）
**字段定义：**
| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | string | 是 | UUID | 笔记唯一ID |
| title | string | 是 | - | 笔记标题，≤50字符 |
| content | text | 是 | - | 笔记内容，≤5000字符 |
| category | string | 是 | "其他" | 分类标签 |
| created_at | datetime | 是 | NOW() | 创建时间 |
| updated_at | datetime | 是 | NOW() | 更新时间 |

**数据验证：**
- 标题长度≤50字符
- 内容长度≤5000字符
- 分类必须为预设分类之一

#### 实体2：收入记录（Income）
**字段定义：**
| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | string | 是 | UUID | 记录唯一ID |
| amount | decimal | 是 | - | 收入金额，≥0 |
| source | string | 是 | "其他" | 收入来源 |
| date | date | 是 | NOW() | 收入日期 |
| note | string | 否 | - | 备注，≤100字符 |
| created_at | datetime | 是 | NOW() | 创建时间 |

#### 实体3：支出记录（Expense）
**字段定义：**
| 字段名 | 类型 | 必填 | 默认值 | 说明 |

| id | string | 是 | UUID | 记录唯一ID |
| amount | decimal | 是 | - | 支出金额，≥0 |
| category | string | 是 | "其他" | 支出分类 |
| date | date | 是 | NOW() | 支出日期 |
| note | string | 否 | - | 备注，≤100字符 |

### 6.2 数据统计

**统计指标：**
| 指标名称 | 计算方法 | 说明 |
|---------|---------|------|
| 月收入总额 | SUM(income.amount) WHERE month | 本月所有收入之和 |
| 月支出总额 | SUM(expense.amount) WHERE month | 本月所有支出之和 |
| 月余额 | 月收入总额 - 月支出总额 | 本月收支差额 |
| 分类支出占比 | 支出分类金额 / 月支出总额 | 各分类支出比例 |
| 预算使用率 | 月支出总额 / 月预算 × 100% | 预算使用进度 |
